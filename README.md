# Customer-Revenue-Analysis-for-Food-Delivery-Platforms


## 📌 Project Overview

This project focuses on analyzing a food delivery dataset using SQL to uncover meaningful insights related to customer behavior, restaurant performance, and delivery operations. The goal is to simulate real-world business analysis and support data-driven decision-making.

---

## 🎯 Objectives

* Analyze customer ordering behavior and engagement
* Evaluate restaurant performance based on ratings and revenue
* Identify high-value customers and ordering trends
* Measure delivery partner efficiency
* Extract time-based insights from order data

---

## 🗂️ Database Structure

The project is built on multiple relational tables:

* **customers** → Customer details (ID, name, city)
* **orders** → Order details (order_id, customer_id, date, amount)
* **restaurants** → Restaurant details (ID, name, city, rating)
* **deliverypartners** → Delivery partner details
* **orderdelivery** → Mapping between orders and delivery partners
* **deliveryupdates** → Delivery status updates

### 💰 Order & Revenue Insights

### 🚚 Delivery Partner Analysis

### 📅 Behavioral Insights
